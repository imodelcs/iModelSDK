# 标准缩写、首字母缩略词和命名规则

通常，缩写和首字母缩略词应仅在常见、预期且明确的情况下使用。否则，请避免使用它们。

在现有概念的上下文中命名新概念时，一致性应该是首要目标。

## 可接受的缩写

| 缩写 | 单词 |
| --- | --- |
| Db | Database |
| Id | Identifier |

## 首字母缩略词和数字缩略词

| 首字母缩略词 | 短语 |
| --- | --- |
| 1d, 2d, 3d | 一维、二维、三维（注意小写 "d"） |
| Html | HyperText Markup Language |
| EC | Engineering Content |
| IO | Input/Output |
| UI | User Interface |
| Uri | Universal Resource Identifier。参见 http://www.w3.org/TR/uri-clarification。出于上述原因，我们更喜欢使用术语 Uri 而不是 Url/Urn。如果确实是 Urn，则改用 'Code' |
| I18N | Internationalization（国际化） |
| L10N | Localization（本地化） |

## 命名规则

| __不要使用这些__ | __请使用__ |
| --- | --- |
| Desc | Description |
| Geo, Geom | Geometry |
| Uuid | Guid |
| Mngt, Mgmt 等 | Management |
| Url, Urn | Uri |
| Def | Definition |
| Ref | Reference |
