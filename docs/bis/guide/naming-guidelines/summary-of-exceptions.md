# 例外情况汇总

下面的列表包含所有例外的汇总。

| 描述 |
| --- |
| 1d、2d 和 3d 的使用需要特别说明。 |
| "谨慎使用缩写和首字母缩略词" 部分指出，两个字母的首字母缩略词必须两个字母都大写，前提是每个字母代表一个不同的单词。例如，UI（"User Interface"）和 IO（"Input/Output"），但 Db（因为单词是 Database 而不是 Data Base）。1d 是 One Dimensional 的缩写——这是两个单词，应该缩写为 OD 或 1D。 |
| "名称不能以数字开头" 规则指出我们不能以 1d、2d 和 3d 开头。最后，"谨慎使用数字" 规则指出我们应该尽量避免完全使用数字。 |
| "组合术语时，按重要性递增顺序排列" 规则建议应该使用 2dDrawing 而不是 Drawing2d——但这会违反前面的规则。 |
| 所以当涉及 1d、2d、3d 时；只在另一个术语之后使用它们，并始终使用小写 'd'。 |
| 即使看起来更一致，也不要使用 OneDimensional、TwoDimensional 或 ThreeDimensional 这些词。 |
| Class 名称应使用单数形式；除了 Relationship——我们希望 Relationship 名称指示基数。 |
